# GTD + 碎碎锚 融合应用

> 基于高敏感+ADHD+INFP特质打造的任务管理系统

## 核心设计理念

本应用融合了 **GTD时间管理** 和 **碎碎锚粉末化拆解** 两大核心方法论：

- **GTD** - 帮你分类任务：收集箱 → 项目清单 → 等待清单 → Someday
- **碎碎锚** - 帮你拆解任务：大目标 → 里程碑 → 模块 → 5分钟粉末动作

---

## 功能模块

### 1. 收集箱 Inbox

所有想法的收集入口，遵循 GTD 收集原则。

**功能：**
- 快速记录任何想法
- 智能决策树引导分类

**决策流程：**
```
这个想法可执行吗？
  ├─ 否 → 参考资料 / Someday / 回收箱
  └─ 是 → 2分钟内能完成？
            ├─ 是 → 立即执行（标记完成）
            └─ 否 → 这是个项目吗？
                      ├─ 是 → 进入项目清单（粉末化拆解）
                      └─ 否 → 进入等待清单 / 上下文清单
```

### 2. 项目清单 Projects

**核心亮点：粉末化任务拆解**

采用 4 级思维导图层级结构，将大目标拆解为 5 分钟可完成的最小动作。

```
根节点（1个目标）
  └─ 一级里程碑（最多3个）
      └─ 二级模块（每个最多5个）
          └─ 粉末叶子节点（5分钟最小闭环）
```

**拆解规则：**

| 层级 | 名称 | 数量限制 | 示例 |
|------|------|----------|------|
| L0 | 根节点 | 同时1个 | 完成「锚点」应用开发 |
| L1 | 里程碑 | ≤3个 | 后端接口开发完成 |
| L2 | 模块 | ≤5个 | 灵感输入框组件开发 |
| L3 | 粉末动作 | 无上限 | 新建Input.vue文件 |

**粉末节点验收标准：**

1. ✅ 单一动作（不含"和""并""同时"）
2. ✅ 5分钟可完成
3. ✅ 零启动门槛（打开电脑就能做）
4. ✅ 结果可验证（有明确完成标志）
5. ✅ 无后续负担（做完就算成功）

### 3. 等待清单 Waiting

管理等待他人完成的任务。

### 4. Someday 有朝一日

暂时不执行但想保留的任务。

### 5. 日历 Calendar

查看有截止日期的任务。

### 6. 番茄钟 Pomodoro

专注计时工具，25分钟工作 + 5分钟休息。

### 7. 回收箱 Trash

已删除任务的临时存放处，可恢复或彻底删除。

## 温柔设计

### 防焦虑机制

- ❌ 无截止日期输入
- ❌ 无完成率进度条
- ❌ 无逾期红色警告
- ❌ 无失败惩罚

### 温柔引导提示

| 操作阶段 | 提示文案 |
|----------|----------|
| 创建根节点 | 只定一个你真正想完成的终点就好哦，不用想过程、不用想细节✨ |
| 添加里程碑 | 我们先把这3个里程碑拆完、做完好不好？太多阶段会让大脑过载哦✨ |
| 拆解模块 | 不用想具体怎么做哦，只要拆成独立的小模块就好✨ |
| 创建粉末动作 | 记得哦，要小到5分钟就能做完，潜意识才不会抗拒✨ |

## 碎碎锚 vs GTD

| GTD | 碎碎锚（本应用） |
|-----|-----------------|
| 横向分类 | 纵向深挖 |
| 通用方法论 | 专注拆解 |
| 适合所有场景 | 更适合大型、需要细节规划的任务 |
| 收集→整理→组织→计划→执行 | 目标→里程碑→模块→粉末动作 |

**简单理解：**
- GTD 解决"任务放哪"的问题
- 碎碎锚解决"任务怎么做"的问题

本应用将两者融合，既能分类管理，又能粉末化拆解执行。
