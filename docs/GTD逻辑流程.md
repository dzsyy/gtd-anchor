# GTD 逻辑流程

## 一、GTD 工作流

```
收集 → 处理 → 组织 → 回顾 → 执行
```

---

## 二、任务生命周期

### 2.1 收集（Inbox）

**入口**：任何时候都可以往收集箱添加任务

```
念头/想法 → 输入标题 → [Enter] → 进入收集箱
```

**支持**：
- 单条添加
- 批量导入（每行一个）

---

### 2.2 处理（Process）

**入口**：点击收集箱里的任务「处理」按钮

```
1. 可行动吗？
   ├── 否 → 5. 不可行动
   └── 是 → 2. 可以一步搞定吗？
              ├── 否 → 创建项目（进入项目清单）
              └── 是 → 3. 2分钟内能搞定吗？
                        ├── 是 → 立即执行 → 归档
                        └── 否 → 4. 该我做吗？
                                  ├── 该我做 → 执行清单
                                  └── 等别人 → 等待清单
```

**决策选项**：

| 步骤 | 选项 | 目标状态 |
|------|------|----------|
| 1. 可行动？ | 是/否 | - |
| 2. 一步搞定？ | 是/否 | 否 → PROJECT |
| 3. 2分钟？ | 是/否 | 是 → ARCHIVED |
| 4. 该我做？ | 我做/等别人 | 我做 → PROJECT / 等别人 → WAITING |
| 5. 不可行动 | 可能清单/回收箱 | SOMEDAY / TRASH |

---

### 2.3 组织（Organize）

**项目清单**：大任务拆解为树形结构

```
项目（ROOT）
  └── 里程碑（≤3个）
        └── 模块（≤5个）
              └── 粉末（无限）
                    ↓
              完成 → 归档
```

**规则**：
- 根节点 = 项目
- 粉末节点 = 最小可执行单元（≥5分钟）
- 随时可「打回收集箱」重新处理

---

### 2.4 流向图

```
                    ┌─ 归档（完成）
                    │
收集箱 ──处理──→ 项目 ┼─ 执行清单 ──完成──→ 归档
  │              │    │
  │              └──→ 等待清单 ──完成──→ 归档
  │
  ├─ 可能清单 ──激活──→ 项目
  │
  └─ 回收箱 ←──删除─┐
                    │
         ┌─ 恢复──→ 收集箱
         │
         └─ 彻底删除
```

---

## 三、各清单职责

| 清单 | 用途 | 出口 |
|------|------|------|
| 收集箱 | 暂存一切念头 | 处理后分流 |
| 执行清单 | 马上要做的事 | 完成→归档 |
| 项目清单 | 需要多步完成的大任务 | 拆解+完成→归档 |
| 等待清单 | 等别人做的事 | 完成→归档 |
| 可能清单 | 将来某时可能做 | 激活→项目 / 删除 |
| 回收箱 | 临时删除 | 恢复 / 彻底删除 |
| 归档 | 已完成的任务 | - |

---

## 四、核心原则

1. **收集箱要清空**：每天或每两天处理一次
2. **粉末化**：拆到5分钟内可完成为止
3. **不设置截止日期**：无压力，专注当下
4. **随时可回**：任何任务都可以打回重新处理
