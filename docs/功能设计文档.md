# GTD + 碎碎锚 功能设计文档

## 一、系统概述

GTD + 碎碎锚 (Sui Sui Mao) 是一款面向高敏感/ADHD/INFP 人群的任务管理与自我成长陪伴应用。

### 1.1 设计理念

- **温和无压**：无截止日期、无强制提醒、正向反馈
- **粉末化分解**：将大任务拆解为可执行的微小颗粒（5分钟最小行动）
- **锚定灵感**：捕捉转瞬即逝的想法，不让灵感丢失

### 1.2 技术架构

| 层级 | 技术栈 |
|------|--------|
| 后端 | Spring Boot 2.x + MyBatis-Plus + MySQL + Redis |
| 前端 | React 19 + TypeScript + Vite + Tailwind CSS |
| 状态管理 | Zustand |
| 思维导图 | mind-elixir |

### 1.3 交互设计特点

#### 1.3.1 温和交互

| 特点 | 描述 |
|------|------|
| 无截止日期 | 不提供日期选择器，避免时间焦虑 |
| 无强制提醒 | 不弹窗、不红点、不焦虑 |
| 正向反馈 | 完成时显示鼓励，无惩罚性提示 |
| 静默操作 | 操作后无噪点提示，专注当前任务 |

#### 1.3.2 粉末化思维

| 特点 | 描述 |
|------|------|
| 5分钟门槛 | 粉末节点必须可5分钟内完成 |
| 渐进式拆分 | 支持 ROOT→MILESTONE→MODULE→POWDER 多层拆分 |
| 随时可回 | 任意节点可「打回收集箱」，不设限制 |
| 即时添加 | 子任务可直接在节点旁添加，无需弹窗 |

#### 1.3.3 灵感捕捉

| 特点 | 描述 |
|------|------|
| 零门槛记录 | 首页即是输入框，Capture 不等待 |
| 标签分流 | 5分钟/30分钟/长期灵感，自动分流 |
| 一键转化 | 灵感可直接转为任务或成果 |

#### 1.3.4 视觉交互

| 特点 | 描述 |
|------|------|
| 思维导图 | 项目结构可视化，支持拖拽、折叠 |
| 分页展示 | 归档列表渐进加载，避免长列表卡顿 |
| 移动优先 | 侧边栏响应式设计，移动端可展开/收起 |
| 暗色主题 | 默认浅色，支持系统主题切换 |

## 二、GTD 模块

### 2.1 任务状态体系

```
INBOX (收集箱) → 处理流程 → PROJECT/WAITING/SOMEDAY/TRASH/ARCHIVED
```

| 状态 | 说明 |
|------|------|
| INBOX | 收集箱，待处理 |
| PROJECT | 项目清单 |
| WAITING | 等待清单 |
| SOMEDAY | 可能清单 |
| TRASH | 回收箱 |
| ARCHIVED | 归档（已完成任务） |

### 2.2 节点层级体系（碎碎锚扩展）

```
ROOT (0) ──→ MILESTONE (≤3) ──→ MODULE (≤5) ──→ POWDER (无限)
项目根节点   一级里程碑          二级模块          粉末叶子节点
```

- 根节点（项目）可创建 ≤3 个里程碑
- 每个里程碑可创建 ≤5 个模块
- 粉末节点无数量限制，但必须 ≥5 分钟可完成

### 2.3 功能清单

#### 2.3.1 收集箱 (Inbox)

| 功能 | 描述 | 交互特点 |
|------|------|----------|
| 单条添加 | 输入任务标题，Enter 提交 | 输入框回车提交，IME 未确定时不触发 |
| 批量导入 | 弹窗输入，每行一个任务，Ctrl+Enter 快速导入 | 弹窗式输入，Ctrl+Enter 快捷提交，显示预览数量 |
| 处理任务 | 点击「处理」进入 GTD 决策流程 | 逐步引导 Dialog，点击按钮流转 |
| 删除任务 | Dropdown 菜单删除 | 悬停显示操作按钮，Dropdown 菜单 |
| IME 兼容 | 使用 `isComposing` 防止输入法未确认时误提交 | 键盘事件拦截，nativeEvent.isComposing 检测 |

**GTD 决策流程**：
1. 可行动吗？→ 是/否
2. 可一步搞定吗？→ 是/否 → 否进入项目
3. 2分钟内能搞定吗？→ 是→立即执行(归档)/否
4. 该我做吗？→ 该我做→执行/等别人→等待
5. 不可行动→ 可能清单/回收箱
6. 等待谁？→ 输入等待对象

**交互流程图**：
```
输入任务 → [Enter] → 加入收集箱 → [点击处理] → GTD 决策引导 → 分流到各清单
```

#### 2.3.2 执行清单 (Next Actions)

| 功能 | 描述 | 交互特点 |
|------|------|----------|
| 任务列表 | 显示所有 PROJECT 状态任务 | 卡片式列表，悬停显示操作 |
| 打回收集箱 | 将任务移回 INBOX | 按钮操作，状态即时更新 |
| 删除任务 | 移到回收箱 | Dropdown 菜单，二次确认 |

#### 2.3.3 项目清单 (Project List)

| 功能 | 描述 | 交互特点 |
|------|------|----------|
| 树形展示 | ROOT → MILESTONE → MODULE → POWDER | 思维导图可视化，节点可折叠展开 |
| 思维导图 | mind-elixir 渲染项目结构图 | 画布式交互，支持缩放、拖拽、平移 |
| 添加子节点 | 在任意节点下添加子任务 | 节点旁快捷添加按钮，弹窗输入标题 |
| 完成任务 | 粉末节点标记完成（≥5分钟行动验证） | 点击复选框，状态即时更新 |
| 打回收集箱 | 单个节点或整棵树打回 | 单节点按钮，树级别需要确认 |
| 项目折叠/展开 | 支持节点折叠展开 | 点击节点图标展开/收起子节点 |
| 导出图片 | 导出思维导图为图片 | 工具栏按钮，生成 PNG 下载 |

**思维导图交互**：
- 左键拖拽画布移动
- 滚轮缩放
- 节点点击展开/折叠子节点
- 节点悬停显示操作菜单
- 工具栏导出 PNG

#### 2.3.4 等待清单 (Waiting List)

| 功能 | 描述 | 交互特点 |
|------|------|----------|
| 任务列表 | 显示所有 WAITING 状态任务 | 卡片列表，显示等待对象标签 |
| 等待对象 | 显示等待谁的信息 | 节点旁显示等待对象名称 |
| 打回收集箱 | 将任务移回 INBOX | 按钮操作即时生效 |
| 完成任务 | 手动标记完成 | 点击复选框，移至归档 |

#### 2.3.5 可能清单 (Someday List)

| 功能 | 描述 | 交互特点 |
|------|------|----------|
| 任务列表 | 显示所有 SOMEDAY 状态任务 | 卡片式列表 |
| 激活任务 | 激活为 PROJECT | 激活按钮，转入项目清单 |
| 打回收集箱 | 将任务移回 INBOX | 按钮操作即时生效 |
| 删除任务 | 移到回收箱 | Dropdown 菜单 |

#### 2.3.6 回收箱 (Trash)

| 功能 | 描述 | 交互特点 |
|------|------|----------|
| 任务列表 | 显示所有 TRASH 状态任务 | 红色警示风格列表 |
| 彻底删除 | 永久删除 | 按钮+确认，无回退 |
| 恢复任务 | 恢复到 INBOX | 恢复按钮即时生效 |

#### 2.3.7 归档 (Archive)

| 功能 | 描述 | 交互特点 |
|------|------|----------|
| 任务列表 | 显示所有 ARCHIVED 状态任务 | 时间倒序排列 |
| 分页展示 | 初始显示 10 条，「查看更多」展开 | 渐进加载，点击展开更多 |
| 恢复任务 | 恢复到 PROJECT 状态 | 恢复按钮移至项目清单 |
| 删除任务 | 移到回收箱 | 移到回收箱而非彻底删除 |
